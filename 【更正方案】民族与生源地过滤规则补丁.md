> 严重等级：🔴 高（可导致推荐不可填报的院校专业组）

## 一、问题描述

现有系统方案在"步骤3：硬性筛选"中，仅包含科目匹配、批次、位次范围、中外合作办学排除、学生个人排除等过滤条件，**缺失了对学生民族身份和生源地的校验**。

这导致系统可能向非少数民族学生推荐仅限少数民族报考的院校专业组，包括但不限于：少数民族预科班、民族班、定向西藏就业计划等。学生若按此预案填报，相关志愿将直接作废，浪费宝贵的志愿名额。

### 1.1 影响范围（数据实测）

对专家版数据 `2026四川高考志愿--专家版合并整理v2.xlsx` 全量扫描，发现民族限制相关记录共 **381条**，涉及 **140所院校**，具体分布：

| 类型 | 记录数 | 涉及院校数 | 典型识别关键词 |
|------|--------|-----------|--------------|
| 少数民族预科 | 277 | 118 | 专业名含"少数民族预科"、"民族预科" |
| 民族班 | 84 | 20 | 专业名含"民族班" |
| 定向西藏 | 27 | 7 | 专业名/备注含"定向西藏"、"定向就业" |
| 本科批(预科)批次 | 104 | — | 批次字段值为"本科批(预科)" |

> 注：以上类别存在交叉重叠，去重后约 **350+ 条唯一记录**。

### 1.2 学生名单现状

当前 `模拟志愿学生名单.xlsx` 共32列，**不包含"民族"和"生源地"字段**。

**默认规则：未注明民族的学生，一律视为汉族（非少数民族）。**

因此，当前名单中的王润和吴欣怡均应按非少数民族处理，必须排除所有民族限制类专业组。

## 二、更正方案

### 2.1 学生名单结构补充（建议）

在 `模拟志愿学生名单.xlsx` 中新增以下两列（插入在"全省位次"列之后）：

| 新增列名 | 数据类型 | 默认值 | 说明 |
|---------|---------|--------|------|
| 民族 | 文本 | 汉族 | 若为少数民族需注明具体民族名称 |
| 生源地 | 文本 | 四川 | 省级行政区名称，影响定向类和地方专项 |

**缺省处理：** 若该字段为空或不存在，系统按"汉族"+"四川"处理。

### 2.2 硬性过滤规则补丁

在系统方案 **§6 筛选规则细化 → 步骤3：硬性筛选** 的现有6条规则之后，新增第7、8条：

---

**规则7：民族身份匹配过滤**

```
IF 学生.民族 == "汉族" OR 学生.民族 为空:
    排除满足以下任一条件的记录：
    ├─ 条件A：专业名称(Col5) 包含 ["少数民族预科", "民族预科", "民族班"]
    ├─ 条件B：批次(Col14) 包含 "预科"
    ├─ 条件C：院校备注(Col5) 或 专业备注(Col10) 包含 ["少数民族", "民族预科"]
    └─ 条件D：专业名称(Col5) 包含 "预科" 且 不含 "预科教育"（排除师范类预科教育专业的误伤）
ELSE:
    保留所有记录（少数民族学生可报预科）
```

**规则8：定向就业/生源地匹配过滤**

```
IF 学生.生源地 != "西藏":
    排除满足以下条件的记录：
    ├─ 条件A：专业名称(Col5) 包含 ["定向西藏", "西藏定向"]
    └─ 条件B：院校备注(Col5) 或 专业备注(Col10) 包含 "定向西藏就业"

IF 学生.生源地 != "新疆":
    排除满足以下条件的记录：
    ├─ 条件A：专业名称(Col5) 包含 ["内地新疆班", "新疆定向"]
    └─ 条件B：院校备注 或 专业备注 包含 "新疆协作计划"
```

### 2.3 关键词匹配清单

以下为完整的过滤关键词清单，按匹配目标分组：

**A组 — 民族限制类（非少数民族必须排除）：**

| 匹配字段 | 关键词 | 匹配方式 |
|---------|--------|---------|
| 专业名称 (Col5) | 少数民族预科 | 包含 |
| 专业名称 (Col5) | 民族预科 | 包含 |
| 专业名称 (Col5) | 民族班 | 包含 |
| 批次 (Col14) | 预科 | 包含 |
| 院校备注 / 专业备注 | 少数民族 | 包含 |
| 院校备注 / 专业备注 | 仅招少数民族 | 包含 |

**B组 — 定向生源地类（非对应生源地必须排除）：**

| 专业名称 / 备注 | 定向新疆 | 生源地≠新疆时排除 |
| 专业名称 / 备注 | 内地新疆班 | 生源地≠新疆时排除 |
| 专业名称 / 备注 | 新疆协作计划 | 生源地≠新疆时排除 |

**C组 — 白名单（不应被误排除）：**

| 关键词 | 说明 |
|--------|------|
| 预科教育 | 师范类专业方向，面向所有考生 |
| 民族学 | 学科名称，非限制条件 |
| 民族音乐学 | 学科名称，非限制条件 |
| 中国少数民族语言文学 | 学科名称，非限制条件（但需确认是否有民族限制） |

### 2.4 代码实现要点

在 `generate_plan_v2.py` 的硬性筛选阶段，新增过滤函数：

```python
# ===== 民族/生源地过滤关键词配置 =====

MINORITY_KEYWORDS = ["少数民族预科", "民族预科", "民族班"]
MINORITY_REMARKS = ["少数民族", "仅招少数民族", "民族预科"]
YUKEA_BATCH_KW = "预科"
WHITELIST_KEYWORDS = ["预科教育", "民族学", "民族音乐", "民族理论"]

DIRECTED_RULES = {
    "西藏": ["定向西藏", "西藏定向", "内地西藏班"],
    "新疆": ["定向新疆", "新疆定向", "内地新疆班", "新疆协作计划"],
}


def should_exclude_by_ethnicity(row, student_ethnicity="汉族"):
    """
    民族身份过滤：非少数民族学生排除民族限制类专业

    参数:
        row: 数据行 (dict or list)
        student_ethnicity: 学生民族，默认"汉族"
    返回:
        (bool, str) - (是否排除, 排除原因)
    """
    if student_ethnicity != "汉族" and student_ethnicity:
        return False, ""

    major_name = str(row.get("专业名称", "") or "")
    batch = str(row.get("批次", "") or "")
    remarks = str(row.get("院校备注", "") or "") + str(row.get("专业备注", "") or "")

    # 白名单保护：如果专业名命中白名单，跳过排除
    for wk in WHITELIST_KEYWORDS:
        if wk in major_name:
            return False, ""

    # 检查专业名称
    for kw in MINORITY_KEYWORDS:
        if kw in major_name:
            return True, f"民族限制：专业名含'{kw}'"

    # 检查批次
    if YUKEA_BATCH_KW in batch:
        return True, f"民族限制：批次含'{YUKEA_BATCH_KW}'"

    # 检查备注
    for kw in MINORITY_REMARKS:
        if kw in remarks:
            return True, f"民族限制：备注含'{kw}'"

    return False, ""


def should_exclude_by_origin(row, student_origin="四川"):
    """
    生源地/定向过滤：非目标生源地学生排除定向类专业

    参数:
        row: 数据行
        student_origin: 学生生源地省份，默认"四川"
    返回:
        (bool, str) - (是否排除, 排除原因)
    """
    major_name = str(row.get("专业名称", "") or "")
    remarks = str(row.get("院校备注", "") or "") + str(row.get("专业备注", "") or "")
    search_text = major_name + remarks

    for region, keywords in DIRECTED_RULES.items():
        if student_origin == region:
            continue  # 生源地匹配，不排除
        for kw in keywords:
            if kw in search_text:
                return True, f"定向限制：含'{kw}'但生源地为{student_origin}"

    return False, ""
```

**集成位置：** 在现有硬性筛选循环内，科目匹配和批次筛选之后、位次范围筛选之前执行：

```python
# 在 filter_candidates() 函数中，现有筛选逻辑之后追加：
excluded_eth, reason_eth = should_exclude_by_ethnicity(row, student.get("民族", "汉族"))
if excluded_eth:
    continue  # 跳过该记录

excluded_org, reason_org = should_exclude_by_origin(row, student.get("生源地", "四川"))
if excluded_org:
    continue  # 跳过该记录
```

### 2.5 结果表标注增强

## 三、对现有预案的影响评估

### 3.1 王润（462分，位次58853，汉族/四川）

预计从当前55组推荐中移除的记录：需实际运行确认，但根据数据分布，在其位次范围（42853~68853）内，少数民族预科和定向类记录预计占 **3~8组**。移除后触发补位算法填充至55组。

### 3.2 吴欣怡（472.5分，位次52608，汉族/四川）

同理，预计移除 **2~6组** 民族限制类记录，由补位算法自动填充。

### 3.3 整体影响

- 过滤发生在硬性筛选阶段（最前端），不影响后续评分、分梯度、去重等流程
- 补位算法已有gap-filling机制，移除后自动补充，最终仍保证55组输出
- 推荐质量提升：所有推荐组均为学生可实际填报的有效志愿

## 五、A3横版打印布局约束

### 5.1 页面基础设置

所有工作表统一采用A3横向打印，通过openpyxl设置：

```python
from openpyxl.worksheet.page import PageMargins
from openpyxl.worksheet.properties import PageSetupProperties

# 纸张与方向
ws.page_setup.paperSize = 8          # A3 (297×420mm)
ws.page_setup.orientation = 'landscape'

# 缩放策略：宽度适配一页，高度不限（自然分页）
ws.page_setup.fitToWidth = 1
ws.page_setup.fitToHeight = 0        # 0 = 不限页数
ws.sheet_properties.pageSetUpPr = PageSetupProperties(fitToPage=True)

# 窄边距（最大化打印区域）
ws.page_margins = PageMargins(
    left=0.3, right=0.3,             # 约7.6mm
    top=0.4, bottom=0.4,             # 约10mm
    header=0.2, footer=0.2           # 约5mm
)

# 水平居中打印
ws.print_options.horizontalCentered = True
```
