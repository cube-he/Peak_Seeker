## 🗄️ 核心数据表设计

### 1. 院校表 (universities)

```sql
CREATE TABLE universities (
    id SERIAL PRIMARY KEY,

    -- 基本信息
    name VARCHAR(200) NOT NULL,                    -- 院校名称
    code VARCHAR(50) UNIQUE,                       -- 院校代码
    province VARCHAR(50),                          -- 院校省份
    city VARCHAR(100),                             -- 院校城市

    -- 院校属性
    university_type VARCHAR(50),                   -- 院校类型（综合/理工/师范等）
    university_level VARCHAR(50),                  -- 院校层次（本科/专科）
    running_level VARCHAR(50),                     -- 办学等级
    running_nature VARCHAR(50),                    -- 办学性质（公办/民办）

    -- 院校标签
    is_double_first_class BOOLEAN DEFAULT FALSE,   -- 是否双一流
    is_985 BOOLEAN DEFAULT FALSE,                  -- 是否985
    is_211 BOOLEAN DEFAULT FALSE,                  -- 是否211
    university_tags TEXT[],                        -- 院校标签（数组）
    university_grade VARCHAR(50),                  -- 院校层级

    -- 学科建设
    discipline_evaluation_level VARCHAR(50),       -- 学科评估等级
    has_master_program BOOLEAN DEFAULT FALSE,      -- 是否有硕士点
    has_doctoral_program BOOLEAN DEFAULT FALSE,    -- 是否有博士点
    master_program_count INTEGER,                  -- 硕士点数量
    master_programs TEXT[],                        -- 硕士点专业
    doctoral_program_count INTEGER,                -- 博士点数量
    doctoral_programs TEXT[],                      -- 博士点专业

    -- 其他属性
    campus_area DECIMAL(10,2),                     -- 校园面积
    transfer_difficulty VARCHAR(50),               -- 转专业难度
    military_training_duration VARCHAR(50),        -- 军训时长
    is_featured BOOLEAN DEFAULT FALSE,             -- 是否特色专业

    -- 备注
    notes TEXT,                                    -- 院校备注

    -- 元数据
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    -- 索引
    INDEX idx_province (province),
    INDEX idx_city (city),
    INDEX idx_type (university_type),
    INDEX idx_double_first_class (is_double_first_class),
    INDEX idx_985 (is_985),
    INDEX idx_211 (is_211)
);
```

### 2. 专业表 (majors)

```sql
CREATE TABLE majors (
    id SERIAL PRIMARY KEY,

    -- 基本信息
    name VARCHAR(200) NOT NULL,                    -- 专业名称
    code VARCHAR(50),                              -- 专业代码

    -- 专业分类
    major_category VARCHAR(100),                   -- 专业大类
    major_level VARCHAR(50),                       -- 专业层次（本科/专科）
    discipline VARCHAR(100),                       -- 学科门类

    -- 专业属性
    major_type VARCHAR(50),                        -- 专业类型
    major_notes TEXT,                              -- 专业备注
    is_restricted BOOLEAN DEFAULT FALSE,           -- 是否受限专业

    -- 就业信息（待补充）
    employment_rate DECIMAL(5,2),                  -- 就业率
    avg_salary INTEGER,                            -- 平均薪资
    employment_direction TEXT,                     -- 就业方向

    -- 元数据
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    -- 索引
    INDEX idx_category (major_category),
    INDEX idx_level (major_level),
    INDEX idx_discipline (discipline)
);
```

### 3. 招生计划表 (enrollment_plans)

```sql
CREATE TABLE enrollment_plans (
    id SERIAL PRIMARY KEY,

    -- 关联信息
    university_id INTEGER NOT NULL REFERENCES universities(id),
    major_id INTEGER NOT NULL REFERENCES majors(id),

    -- 招生年份和省份
    year INTEGER NOT NULL,                         -- 招生年份
    province VARCHAR(50) NOT NULL,                 -- 招生省份

    -- 招生计划
    plan_count INTEGER,                            -- 计划招生人数
    plan_notes TEXT,                               -- 计划备注

    -- 招生要求
    batch VARCHAR(50),                             -- 批次（本科一批/二批等）
    level VARCHAR(50),                             -- 层次
    subjects VARCHAR(100),                         -- 科目要求
    subject_requirements TEXT,                     -- 选考要求
    duration VARCHAR(20),                          -- 学制
    tuition INTEGER,                               -- 学费
    is_sino_foreign BOOLEAN DEFAULT FALSE,         -- 是否中外合作

    -- 元数据
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    -- 索引
    INDEX idx_year (year),
    INDEX idx_province (province),
    INDEX idx_university (university_id),
    INDEX idx_major (major_id),
    INDEX idx_batch (batch),
    UNIQUE INDEX idx_unique_plan (university_id, major_id, year, province)
);
```

### 4. 录取数据表 (admission_records)

```sql
CREATE TABLE admission_records (
    id SERIAL PRIMARY KEY,

    -- 关联信息
    university_id INTEGER NOT NULL REFERENCES universities(id),
    major_id INTEGER NOT NULL REFERENCES majors(id),

    -- 年份和省份
    year INTEGER NOT NULL,                         -- 录取年份
    province VARCHAR(50) NOT NULL,                 -- 录取省份

    -- 专业录取数据
    major_min_score INTEGER,                       -- 专业最低投档分
    major_min_rank INTEGER,                        -- 专业最低投档分位次
    major_admission_count INTEGER,                 -- 专业录取人数

    -- 院校整体录取数据
    university_min_score INTEGER,                  -- 院校最低投档分
    university_min_rank INTEGER,                   -- 院校最低投档分位次
    university_avg_score INTEGER,                  -- 院校平均分
    university_avg_rank INTEGER,                   -- 院校平均位次
    university_max_score INTEGER,                  -- 院校最高分
    university_max_rank INTEGER,                   -- 院校最高位次
    university_admission_count INTEGER,            -- 院校录取人数

    -- 元数据
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

    -- 索引
    INDEX idx_year (year),
    INDEX idx_province (province),
    INDEX idx_university (university_id),
    INDEX idx_major (major_id),
    INDEX idx_major_min_score (major_min_score),
    INDEX idx_major_min_rank (major_min_rank),
    UNIQUE INDEX idx_unique_record (university_id, major_id, year, province)
);
```

    df = pd.read_excel('2025高考志愿填报--西典学校整合版.xlsx', sheet_name='四川')

    # 连接数据库
    conn = psycopg2.connect(
        host="localhost",
        database="volunteer_helper",
        user="postgres",
        password="password"
    )
    cur = conn.cursor()

    # 1. 导入院校数据（去重）

                       '院校类型', '院校层次', '办学性质', '是否双一流']].drop_duplicates()

    for _, row in universities.iterrows():
        cur.execute("""
            INSERT INTO universities (name, code, province, city, university_type,

## 🔍 常用查询示例

### 1. 根据分数和位次查询可报考院校
```sql
SELECT DISTINCT
    u.name AS university_name,
    u.province,
    u.is_double_first_class,
    ar.major_min_score,
    ar.major_min_rank,
    m.name AS major_name
FROM admission_records ar
JOIN universities u ON ar.university_id = u.id
JOIN majors m ON ar.major_id = m.id
WHERE ar.year = 2024
  AND ar.province = '四川'
  AND ar.major_min_rank >= :user_rank - 10000  -- 冲
  AND ar.major_min_rank <= :user_rank + 5000   -- 保
ORDER BY ar.major_min_rank ASC;
```

### 2. 查询院校历年录取趋势
```sql
SELECT
    year,
    university_min_score,
    university_min_rank,
    university_avg_score,
    university_max_score
FROM admission_records
WHERE university_id = :university_id
  AND major_id = :major_id
  AND province = '四川'
ORDER BY year DESC;
```

### 3. 查询专业就业前景
```sql
SELECT
    m.name AS major_name,
    m.employment_rate,
    m.avg_salary,
    COUNT(DISTINCT ar.university_id) AS university_count
FROM majors m
LEFT JOIN admission_records ar ON m.id = ar.major_id
WHERE m.major_category = :category
GROUP BY m.id, m.name, m.employment_rate, m.avg_salary
ORDER BY m.avg_salary DESC;
```

## 🔐 数据安全

### 敏感数据处理
1. **用户密码**：使用 bcrypt 加密存储
2. **个人信息**：加密存储，访问日志记录
3. **支付信息**：不存储完整支付信息，仅存订单号
4. **数据备份**：每日自动备份，异地存储

### 数据访问控制
1. **角色权限**：管理员、VIP用户、普通用户
2. **API限流**：防止数据爬取
3. **审计日志**：记录所有数据访问
