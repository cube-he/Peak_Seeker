## 三、梯度划分策略（6段细分）

### 3.1 梯度区间定义（v4.0 系统抽样法）

> **v4.0变更说明**：v3.0虽然采用绝对偏移法（P-16000~P+10000）确定了合理的总范围，但在每个梯度内选取志愿时仍存在"扎堆"问题——同一梯度内多个志愿位次相近，梯度没有真正拉开。v4.0引入**系统抽样法（Systematic Sampling）**，将整个位次范围等距分成55个小区间（bin），每个bin选1个最优院校专业组（由锚定专业驱动），确保55个志愿组在位次轴上均匀散开。

#### 3.1.1 总体范围

以学生全省预估位次（P）为锚点，通过自适应算法（详见14.3）确定冲端偏移 `range_up` 和保端偏移 `range_down`，总范围 `P-range_up ~ P+range_down`。

默认情况（P≈55000）：range_up=16000, range_down=10000，总跨度26000。高分/低分段自动缩放。

#### 3.1.2 系统抽样核心逻辑（动态bin）

**核心原则：可多不可少，至少55个院校专业组，梯度必须均匀散开。**

```
步骤1【确定初始bin跨度】
  初始bin跨度 = 26000 ÷ 55 ≈ 473个位次
  生成bin列表: bin_i = [P-16000 + i×473, P-16000 + (i+1)×473)

步骤2【填充bin】
  对每个bin，从备选池中筛选位次落在该区间的所有候选

步骤3【每bin选锚定专业→映射到院校专业组】
  综合评分选出最优专业（锚定专业），其所在院校专业组即为该bin的志愿
  （两层选取详见14.11）

步骤4【动态调整——确保≥55个院校专业组】
  统计有效picks（非空bin数量，去重后的院校专业组数）
  如果 有效picks ≥ 55 → 完成
  如果 有效picks < 55 → 缩小bin跨度，重新执行步骤1-3
    缩小策略: bin跨度 = bin跨度 × 0.8（每轮缩20%）
    重复直到 有效picks ≥ 55 或 bin跨度 < 100（下限保护）

步骤5【标注梯度标签——比例化边界（v4.3）】
  梯度边界不再硬编码，按range_up和range_down的比例动态计算（详见14.10）：
    位次 ∈ [P-range_up,          P-range_up*0.625)  → 高冲   (冲端上部62.5%)
    位次 ∈ [P-range_up*0.625,    P-range_up*0.3125) → 冲刺   (冲端下部31.25%)
    位次 ∈ [P-range_up*0.3125,   P)                 → 稳偏冲 (冲端尾部31.25%)
    位次 ∈ [P,                   P+range_down*0.4)  → 稳妥   (保端上部40%)
    位次 ∈ [P+range_down*0.4,    P+range_down*0.7)  → 保偏稳 (保端中部30%)
    位次 ∈ [P+range_down*0.7,    P+range_down]      → 保底   (保端下部30%)

  默认值(range_up=16000, range_down=10000)时，等价于原来的固定值：
    高冲=[P-16000, P-10000), 冲刺=[P-10000, P-5000), 稳偏冲=[P-5000, P)
    稳妥=[P, P+4000), 保偏稳=[P+4000, P+7000), 保底=[P+7000, P+10000]
```

> **为什么"可多不可少"**：这是预案，多几个院校专业组给家长和学生更多讨论空间。如果bin跨度缩小导致选出60、65个组也完全可以，人工干预时可以再精简。

#### 3.1.3 综合评分规则（分层优先级 + 动态权重滑动）

**基础优先级：院校层级（或排名） > 办学性质 > 专业（专业名称+排名） > 其他**

权重系数适度拉开，不极端碾压。高层维度在多数情况下优先，但当高层维度得分接近时，低层维度的差异可以影响最终排序——这更符合实际决策。

**评分公式：**
```
总分 = 院校层级得分 × W1(t)
     + 办学性质得分 × W2(t)
     + 专业得分     × W3(t)
     + 其他得分     × W4(t)

其中 t = bin_index / total_bins, t ∈ [0, 1]
     t=0 为最冲端, t=1 为最保端
     W1 > W2 > W3 > W4 始终成立（动态滑动不改变顺序）
```

**第一层：院校层级/排名（基础权重最高）**

| 原始分 | 规则 |
|--------|------|
| 5 | 985 |
| 4 | 211（非985） |
| 3 | 双一流（非211） |
| 2 | 省重点/省属重点 |
| 1 | 普通本科 |

权重W1：冲端(t=0)=5.0，保端(t=1)=3.5。冲端更看重院校层级差距，保端层级仍然优先但差距稍缩。

**第二层：办学性质（基础权重次之）**

| 原始分 | 规则 |
|--------|------|
| 3 | 公办 |
| 0 | 民办 |

权重W2：冲端=3.0，保端=3.5。全程重要，保端略升（兜底必须公办）。

**第三层：专业（专业名称 + 排名，参照《推荐与慎选清单》）**

专业评分综合两个维度：专业名称本身的前景（对照推荐/慎选清单） + 该校该专业的排名（学科评估+软科评级）。

| 子维度 | 原始分规则 |
|--------|-----------|
| 专业推荐度 | 推荐清单中的专业=+3，中性专业=+1，慎选清单中的专业=0 |
| 专业排名 | 学科评估A类=+4, B类=+3, C类=+2, 软科A级(无学科评估)=+2, 无评级=0 |
| 专业得分 | = 专业推荐度 + 专业排名（满分7） |

> **专业推荐度**依据 `【参考】2026高考专业推荐与慎选清单.xlsx` 中的推荐专业(AI视角)和慎选专业(AI视角)两个sheet匹配。

权重W3：冲端=2.0，保端=3.0。冲端专业让步于院校（冲名校时专业可以略妥协），保端专业权重提升（兜底要保好专业）。

**第四层：其他因素（辅助加分）**

| 子维度 | 原始分规则 | 满分 |
|--------|-----------|------|
| 招生计划 | ≥10人=3, ≥5人=2, <5人=1 | 3 |
| 保研率 | ≥10%=3, ≥5%=2, ≥1%=1, 无=0 | 3 |
| 地域匹配 | 符合偏好城市=2, 不符=0 | 2 |
| 其他得分 | = 招生计划分 + 保研率分 + 地域分（满分8） | 8 |

权重W4：冲端=1.5，保端=2.5。保端辅助因素更重要（计划大、地域好=确定性高）。

**权重系数汇总与滑动：**

| 层 | 维度 | W冲端(t=0) | W中间(t=0.5) | W保端(t=1) | 满分×W范围 |
|----|------|-----------|-------------|-----------|-----------|
| 1 | 院校层级 | 5.0 | 4.25 | 3.5 | 17.5~25.0 |
| 2 | 办学性质 | 3.0 | 3.25 | 3.5 | 0~10.5 |
| 3 | 专业 | 2.0 | 2.5 | 3.0 | 0~21.0 |
| 4 | 其他 | 1.5 | 2.0 | 2.5 | 0~20.0 |

> 层级间有区分但不极端：院校层级满分得分≈25，专业满分得分≈21，同一量级内有区分。当院校层级相同时，专业好坏可以决定排序；但通常高一级的院校仍优于低一级。

**线性插值公式：**
```python
t = bin_index / total_bins

def W(t, w_start, w_end):
    return w_start + (w_end - w_start) * t

def score(candidate, t):
    # 第一层：院校层级
    L1 = tier_score(candidate) * W(t, 5.0, 3.5)   # 原始分1~5

    # 第二层：办学性质
    L2 = (3 if is_public(candidate) else 0) * W(t, 3.0, 3.5)

    # 第三层：专业（推荐度 + 排名）
    major_rec = recommend_score(candidate)  # 推荐清单=3, 中性=1, 慎选=0
    major_rank = discipline_score(candidate) # A类=4, B类=3, C类=2, 无=0
    L3 = (major_rec + major_rank) * W(t, 2.0, 3.0)

    # 第四层：其他
    other = plan_score(candidate) + postgrad_score(candidate) + location_score(candidate)
    L4 = other * W(t, 1.5, 2.5)

    return L1 + L2 + L3 + L4 + student_bonus(candidate)
```

**效果验证：**

*场景1 高冲端(t=0)——同为公办，985弱专业 vs 省重点强专业：*
- 985+公办+慎选专业+无评级 → 5×5.0 + 3×3.0 + (0+0)×2.0 + 5×1.5 = 41.5
- 省重点+公办+推荐专业+A类 → 2×5.0 + 3×3.0 + (3+4)×2.0 + 5×1.5 = 40.5
- → 985胜出，冲端院校层级优先 ✓（但差距不大，说明极端情况下强专业可以接近）

*场景2 保底端(t=1)——同层级，好专业+大计划 vs 差专业+小计划：*
- 省重点+公办+推荐B类+计划15人 → 2×3.5 + 3×3.5 + (3+3)×3.0 + (3+1+2)×2.5 = 50.5
- 省重点+公办+慎选无评级+计划3人 → 2×3.5 + 3×3.5 + (0+0)×3.0 + (1+0+0)×2.5 = 20.0
- → 好专业+大计划大幅胜出 ✓

#### 3.1.3a 优先级模式配置（按学生需求切换）

评分模型支持两种优先级模式，通过 `priority_mode` 参数在生成时指定：

**模式A：院校优先（默认）**
适用于：无明确专业要求，或以院校平台/考研/考公为目标的学生
```
优先级：院校层级 > 办学性质 > 专业 > 其他
权重分配：W1(院校)=5.0~3.5, W2(性质)=3.0~3.5, W3(专业)=2.0~3.0, W4(其他)=1.5~2.5
```

**模式B：专业优先**
适用于：有明确专业方向（如"一定要学临床医学"、"只考虑计算机类"）的学生
```
优先级：专业 > 办学性质 > 院校层级 > 其他
权重分配：W1(专业)=5.0~3.5, W2(性质)=3.0~3.5, W3(院校)=2.0~3.0, W4(其他)=1.5~2.5
```

核心区别：**只是把权重值在维度之间做了交换**，专业拿到最高权重(5.0~3.5)，院校降到第三层(2.0~3.0)。滑动逻辑、原始分规则、其他维度全部不变。

```python
def score(candidate, t, priority_mode='A'):
    tier  = tier_score(candidate)
    nature = 3 if is_public(candidate) else 0
    major = recommend_score(candidate) + discipline_score(candidate)
    other = plan_score(candidate) + postgrad_score(candidate) + location_score(candidate)

    if priority_mode == 'A':  # 院校优先
        return (tier   * W(t, 5.0, 3.5)
              + nature * W(t, 3.0, 3.5)
              + major  * W(t, 2.0, 3.0)
              + other  * W(t, 1.5, 2.5)
              + student_bonus(candidate))

    elif priority_mode == 'B':  # 专业优先
        return (major  * W(t, 5.0, 3.5)   # 专业提到第一层
              + nature * W(t, 3.0, 3.5)   # 办学性质仍第二
              + tier   * W(t, 2.0, 3.0)   # 院校降到第三层
              + other  * W(t, 1.5, 2.5)
              + student_bonus(candidate))
```

**模式B效果验证（t=0 冲端）：**
- 普通本科+公办+推荐A类 → (3+4)×5.0 + 3×3.0 + 1×2.0 + 5×1.5 = 53.5
- 985+公办+慎选无评级 → (0+0)×5.0 + 3×3.0 + 5×2.0 + 5×1.5 = 26.5
- → 专业优先模式下，好专业的普通本科远胜差专业的985 ✓

**模式B滑动逻辑：**
- 冲端(t=0)：博最好的专业（推荐+A类评估权重拉满），学校可以妥协
- 保端(t=1)：专业仍优先但差距缩小，办学性质和招生计划上升，确保录取

**各学生模式配置：**

| 学生 | 模式 | 原因 | 额外偏好 |
|------|------|------|---------|
| 王润 | A（院校优先） | 无明确专业方向，师范偏好通过加分体现 | 师范+5，成都+2 |
| 吴欣怡 | A（院校优先） | 无明确专业方向，排除法律即可 | 法学排除，成都/广深+2 |
| （示例）某学生 | B（专业优先） | "只想学临床医学" | 临床医学+10，其他医学类+5 |

> 后续如有学生提出"专业优先"需求，在学生信息采集阶段标注 `priority_mode=B` 即可，评分系统自动切换。

## 五、2026年四川新高考风险点清单

### 5.1 数据参考价值变化
- 2025是新高考第一年，2026为第二年，25年数据仅一届参考基线有限
- 院校专业组2026年可能重新编组（冷热混编→优化分组→分数回升）
- 选科要求可能调整（如去年"历史+化学"今年改"不限"→分数波动）

### 5.2 AI时代专业风险（2026新增关注）
- 传统会计/翻译/基础编程/基础设计等专业AI替代加速
- 人工智能/集成电路/新能源等方向人才缺口持续扩大
- 选专业需评估"AI抗性"——参见《专业推荐与慎选清单》

### 5.3 分数线波动风险
- 试卷难度变化导致分数大幅波动，**位次才是核心**
- 2025年冷热混编导致部分院校分数偏低，2026年可能大幅回升

### 5.4 细节失误风险
- 选科不符导致无效志愿
- 忽略单科成绩限制/身体条件限制/性别限制
- 志愿扎堆同一分数段导致批量滑档

### 5.5 应对策略
- 以位次为核心，不看绝对分数
- 6段细分梯度拉开，避免扎堆
- 核对招生章程每一个细节
- 备选池预留充足替换空间

### 6.3 综合评分排序规则（v4.0，替代旧的固定优先级）

> v4.0使用分层优先级+动态权重滑动评分系统，详见3.1.3。以下为默认模式A的优先级总览：

```python
def adaptive_range(P):
    """根据学生位次自动计算合理的冲端/保端偏移量"""
    # 冲端：不超过位次的30%，且不超过16000，下限3000
    range_up = max(3000, min(16000, int(P * 0.30)))
    # 保端：不超过位次的20%，且不超过10000，下限2000
    range_down = max(2000, min(10000, int(P * 0.20)))
    # 保底硬边界：不超过本科线尾部预估位次（约180000）
    if P + range_down > 180000:
        range_down = 180000 - P
    return range_up, range_down
```

## 十二、已知问题与解决记录

### 12.1 问题：美化子代理覆盖备选池数据

**发生时间**：2026-04-02

**问题描述**：
对王润v2预案文件进行样式美化时，子代理在处理过程中未正确保留备选池sheet的数据。美化完成后，6个备选池sheet全部为空（0行），文件从原始的603KB缩小到28KB，数据完全丢失。

**根因分析**：
子代理在美化过程中可能重建了sheet结构（先删除再创建），或者在写入样式时以空白sheet覆盖了原有数据。openpyxl在处理大量数据的sheet时，如果操作顺序不当（如先clear再写样式而未回填数据），会导致数据丢失。

**解决方案**：
1. 从源数据 `2026四川高考志愿--专家版合并整理v2.xlsx` 重新筛选各梯度备选池数据
2. 使用学生位次（56668）按6级梯度重新计算位次区间并筛选
3. 筛选后写入新sheet，删除空的原sheet，重命名新sheet为标准命名
4. 重新应用备选池美化样式

**恢复结果**：

| 备选池 | 恢复后行数 |
|--------|-----------|
| 高冲段 | 1,242 |
| 冲刺段 | 364 |
| 稳偏冲段 | 374 |
| 稳妥段 | 506 |
| 保偏稳段 | 729 |
| 保底段 | 2 |

**附加问题**：子代理创建新sheet时使用了不同的sheet名称（如"高冲备选池"而非"【备选池】高冲段"），导致文件内出现13个sheet（6个空原sheet + 6个有数据的新sheet + 1个结果sheet）。需要额外步骤删除空sheet并重命名。

### 12.2 预防措施（写入后续子代理Prompt）

为避免此类问题再次发生，美化子代理的Prompt中应增加以下约束：

1. **数据优先原则**：美化操作必须先读取并缓存所有现有数据，再进行样式修改，最后验证数据完整性
2. **非破坏性操作**：禁止删除或重建sheet，只在现有sheet上修改样式
3. **行数校验**：美化前后对比每个sheet的行数，如果减少则报错中止
4. **sheet命名一致性**：备选池sheet必须使用标准命名格式 "【备选池】{梯度}段"
5. **文件大小校验**：美化后文件大小不应显著小于美化前（允许±20%波动），否则警告数据可能丢失

### 12.3 推荐的美化流程（安全版）

```
Step 1: 读取文件，记录每个sheet名称和行数（作为基线）
Step 2: 用普通模式(非read_only)打开workbook
Step 3: 逐sheet应用样式（不删除/重建任何sheet）
Step 4: 保存到BytesIO缓冲区
Step 5: 重新读取缓冲区，验证每个sheet行数与基线一致
Step 6: 验证通过后写入最终文件路径
Step 7: 输出验证报告（sheet名称 + 行数 + 通过/失败）
```

**预处理步骤（构建匹配词表）**：

```python
def preprocess_list_entry(raw_name):
    """将清单中一条专业名称拆解为匹配规则列表"""
    rules = []

    # Step 1: 提取括号内的限定条件
    qualifier = ""
    base_name = raw_name
    if '(' in raw_name or '（' in raw_name:
        # 统一为半角括号
        normalized = raw_name.replace('（', '(').replace('）', ')')
        match = re.match(r'^(.+?)\((.+?)\)$', normalized)
        if match:
            base_name = match.group(1)
            qualifier = match.group(2)

    # Step 2: 按"/"拆分为多个候选名称
    candidates = [c.strip() for c in base_name.split('/')]

    # Step 3: 解析限定条件
    exclude_condition = None
    if qualifier.startswith('非'):
        # "非师范" → 排除包含"师范"的
        exclude_condition = qualifier[1:]
    elif qualifier and not qualifier.startswith('非'):
        # "5+3一体化" → 仅作为附加匹配提示，不作为排除条件
        # "传统方向" → 无法通过数据判断，忽略限定，全量匹配
        pass

    for c in candidates:
        rules.append({
            'match_keyword': c,         # 用于包含匹配的关键词
            'exclude_keyword': exclude_condition,  # 需要排除的关键词
            'is_class_name': c.endswith('类'),     # 是否用Col7专业类匹配
        })

    return rules
```

**三级匹配执行流程**：

```python
def match_recommend_score(col5_name, col7_class, recommend_rules, caution_rules):
    """
    返回: 3=推荐, 0=慎选, 1=中性
    优先级: 推荐清单 > 慎选清单 > 默认中性
    """
    col5 = str(col5_name).strip()
    col7 = str(col7_class).strip() if col7_class else ""

    # Level 1: 精确匹配（Col5完全等于匹配词）
    for rule in recommend_rules:
        if col5 == rule['match_keyword']:
            return 3
    for rule in caution_rules:
        if col5 == rule['match_keyword']:
            if rule['exclude_keyword'] and rule['exclude_keyword'] in col5:
                continue  # 命中排除条件，跳过
            return 0

    # Level 2: 包含匹配（Col5包含匹配词，且不命中排除条件）
    for rule in recommend_rules:
        if rule['match_keyword'] in col5:
            return 3
    for rule in caution_rules:
        if rule['match_keyword'] in col5:
            if rule['exclude_keyword'] and rule['exclude_keyword'] in col5:
                continue
            return 0

    # Level 3: 专业类匹配（Col7等于清单中以"类"结尾的匹配词）
    for rule in recommend_rules:
        if rule['is_class_name'] and col7 == rule['match_keyword']:
            return 3
    for rule in caution_rules:
        if rule['is_class_name'] and col7 == rule['match_keyword']:
            return 0

    return 1  # 未匹配，中性

```

**冲突处理规则**：
- 同时命中推荐和慎选（如"会计学"既在推荐清单的"会计学/审计学"中，又在慎选的"会计学(传统方向)"中）→ 按**推荐**处理，因为推荐清单的会计学指的是数智会计方向
- 实际执行时不会冲突——推荐清单优先检查，命中即返回，不再查慎选清单

### 14.3 高/低位次学生的自适应范围算法（Issue #9）

**问题**：当前固定范围 P-16000~P+10000 仅适合位次4万~8万的学生。极端情况分析：

| 位次P | P-16000 | P+10000 | 问题 |
|-------|---------|---------|------|
| 2,000 | -14,000(无效) | 12,000 | 冲端变成负数 |
| 8,000 | -8,000(无效) | 18,000 | 冲端进入前100名区域，候选极少 |
| 50,000 | 34,000 | 60,000 | 正常 |
| 150,000 | 134,000 | 160,000 | 保端已接近本科线尾 |

**自适应算法**（已写入第九章全局参数配置表）：

核心思想——冲端偏移量与位次成正比（位次越高冲得越少），保端同理，加硬边界保护：

```python
def adaptive_range(P, manual_up=None, manual_down=None):
    """
    P: 学生位次
    manual_up/down: 手动覆盖值（优先级最高）
    返回: (range_up, range_down)
    """
    if manual_up is not None and manual_down is not None:
        return manual_up, manual_down

    # 基础公式
    range_up = int(P * 0.30)
    range_down = int(P * 0.20)

    # 上下限保护
    range_up = max(3000, min(16000, range_up))
    range_down = max(2000, min(10000, range_down))

    # 保底硬边界：不超过本科线尾部预估位次
    BACHELOR_TAIL = 180000  # 四川本科批B段尾部预估
    if P + range_down > BACHELOR_TAIL:
        range_down = max(2000, BACHELOR_TAIL - P)

    # 冲端硬边界：不低于位次1
    if P - range_up < 1:
        range_up = P - 1

    return range_up, range_down
```

**配置建议**：全局参数表中的 `range_up` 和 `range_down` 改为默认值"auto"，表示使用自适应算法。如果手动指定具体数字则覆盖算法。

### 14.4 去重策略详解（Issue #4）

> **⚠️ v4.4更新**：本节的单层去重策略已被14.11.5的三步去重策略替代。保留原文供参考，执行时以14.11.5为准。

**~~v4.3策略定义：按"院校代码"(Col1)去重，不按"院校+专业组"去重~~**

**v4.4策略定义：双层去重——先按院校专业组(Col3)去重，再按院校代码(Col1)限频。** 详见14.11.5。

v4.3原始理由（已部分修正）：志愿预案的目的是让学生覆盖尽可能多的院校选择。同一院校的不同专业组（例如"四川大学01组"和"四川大学03组"）虽然是不同的投档单位，但对学生来说都是"去四川大学"——如果把5个志愿位都给四川大学的不同专业组，等于放弃了4个院校选择机会。v4.4保留同校最多2个不同专业组的限制，同时新增了同一专业组不重复的约束。

**去重规则**：

```python
def deduplicate_result(picks, max_same_school=2):
    """
    picks: list of (bin_index, candidate) 按bin顺序排列
    max_same_school: 同一院校代码在结果表中最多出现次数
    返回: 去重后的picks
    """
    school_count = {}  # {院校代码: [已选的(bin_idx, candidate)列表]}
    final_picks = []
    replacements_needed = []

    for bin_idx, cand in picks:
        code = cand.col1_school_code
        if code not in school_count:
            school_count[code] = []

        if len(school_count[code]) < max_same_school:
            school_count[code].append((bin_idx, cand))
            final_picks.append((bin_idx, cand))
        else:
            # 超出限额，需要替换
            # 比较：当前候选 vs 已选中的最低分候选
            existing = school_count[code]
            worst_existing = min(existing, key=lambda x: x[1].total_score)
            if cand.total_score > worst_existing[1].total_score:
                # 当前候选更优，替换掉最差的那个
                replacements_needed.append(worst_existing[0])  # 被替换的bin需要重选
                existing.remove(worst_existing)
                existing.append((bin_idx, cand))
                final_picks.remove(worst_existing)
                final_picks.append((bin_idx, cand))
            else:
                # 当前候选不如已有的，当前bin需要重选
                replacements_needed.append(bin_idx)

    # 对需要重选的bin，取该bin内排名第2的候选（排除已在结果表中的院校）
    for bin_idx in replacements_needed:
        bin_pool = get_bin_candidates(bin_idx)  # 该bin的全部候选
        for alt in bin_pool:
            alt_code = alt.col1_school_code
            if alt_code not in school_count or len(school_count[alt_code]) < max_same_school:
                final_picks.append((bin_idx, alt))
                school_count.setdefault(alt_code, []).append((bin_idx, alt))
                break
        # 如果该bin内所有候选的院校都已满额，该bin留空（允许）

    return sorted(final_picks, key=lambda x: x[0])
```

**备选池不去重**：备选池保留所有符合条件的候选，不做去重。去重只在结果表（Sheet 1）层面执行。

### 14.5 同一院校多专业组的限频策略（Issue #10）

> **⚠️ v4.4更新**：本节的限频策略已整合进14.11.5的三步去重策略（Step 2 + Step 3）。保留原文供参考，执行时以14.11.5为准。

**问题**：数据源中同一学校可能有10+个专业组，如果这些专业组的位次接近，可能落在相邻的多个bin中，导致结果表中连续多个志愿都是同一学校。

**限频规则（在去重之上叠加）**：

```
规则1: 全局限额——同一院校代码在结果表中最多出现 max_same_school 次（默认2次）
规则2: 局部限频——同一院校代码在连续 locality_window（默认3）个bin中最多被选1次
```

**执行顺序**：步骤5c选最优 → 步骤5e全局去重（14.4） → 步骤5f局部限频

```python
def locality_check(final_picks, locality_window=3):
    """检查并修复局部扎堆"""
    picks = list(final_picks)  # 按bin_index排序

    for i in range(len(picks)):
        bin_idx_i, cand_i = picks[i]
        code_i = cand_i.col1_school_code

        # 检查前window个已选志愿中是否有同校
        conflict = False
        for j in range(max(0, i - locality_window + 1), i):
            if picks[j][1].col1_school_code == code_i:
                conflict = True
                break

        if conflict:
            # 当前bin换成次优候选（不同院校）
            bin_pool = get_bin_candidates(bin_idx_i)
            for alt in bin_pool:
                if alt.col1_school_code != code_i:
                    picks[i] = (bin_idx_i, alt)
                    break

    return picks
```

**全局参数新增**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `locality_window` | 3 | 局部限频窗口大小（连续N个bin内同一院校最多1次） |

**数据现状**（基于实际数据验证）：

**处理方案——三类分流**：

```python
def classify_new_major(candidate):
    """对新增专业分类处理"""
    has_col25 = candidate.col25 is not None and str(candidate.col25).strip() != ''
    has_col28 = candidate.col28 is not None and str(candidate.col28).strip() != ''
    has_col38 = candidate.col38 is not None and str(candidate.col38).strip() != ''
    is_new = str(candidate.col15).strip() == '新增'

    if not is_new:
        # 非新增专业，正常处理
        return 'NORMAL'

    if has_col25:
        # 新增专业但有组级位次 → 用Col25参与bin分配，但降低评分置信度
        return 'NEW_WITH_GROUP_DATA'

    if has_col38:
        # 新增专业，无25年数据但有24年数据 → 用Col38参与bin分配
        return 'NEW_WITH_24_DATA'

    # 全无数据
    return 'NEW_NO_DATA'
```

**各类型在bin分配中的处理**：

| 类型 | bin分配位次来源 | 评分调整 | 风险提示 | 备选池处理 |

| NORMAL | Col25→Col28→Col38 | 无调整 | 无额外提示 | 正常参与 |
| NEW_WITH_GROUP_DATA | Col25(组级位次) | 评分×0.9（打9折降低排名） | "新增专业，仅有组级位次参考" | 正常参与，标注"新增" |
| NEW_WITH_24_DATA | Col38(24年位次) | 评分×0.85 | "仅参考24年数据，25年无录取记录" | 正常参与，标注"仅24" |
| NEW_NO_DATA | 不参与bin分配 | — | — | 保留在备选池末尾，单独分组"▶ 无历史数据专业"，供人工参考 |

### 14.10 梯度边界比例化算法（v4.3新增）

**问题**：当自适应算法将 range_up 从16000缩小到4500（如P=15000的高分段学生），原来的固定梯度边界（P-16000=高冲起点、P-10000=冲刺起点等）完全失效。需要将梯度边界从绝对值改为相对比例。

**比例系数推导**（从原始默认值反推）：

```
原始冲端划分（range_up=16000）：
  高冲 = [P-16000, P-10000)  → 起止比例 = [1.0, 0.625) of range_up
  冲刺 = [P-10000, P-5000)   → 起止比例 = [0.625, 0.3125) of range_up
  稳偏冲 = [P-5000, P)       → 起止比例 = [0.3125, 0) of range_up

原始保端划分（range_down=10000）：
  稳妥 = [P, P+4000)         → 起止比例 = [0, 0.4) of range_down
  保偏稳 = [P+4000, P+7000)  → 起止比例 = [0.4, 0.7) of range_down
  保底 = [P+7000, P+10000]   → 起止比例 = [0.7, 1.0] of range_down
```

**算法实现**：

```python
# 比例常量（从默认值反推，固定不变）
GRADIENT_RATIOS = {
    # 冲端各段占range_up的比例边界（从远到近）
    'rush_splits': [1.0, 0.625, 0.3125, 0.0],  # 高冲|冲刺|稳偏冲
    # 保端各段占range_down的比例边界（从近到远）
    'safe_splits': [0.0, 0.4, 0.7, 1.0],        # 稳妥|保偏稳|保底
}

GRADIENT_LABELS = ['高冲', '冲刺', '稳偏冲', '稳妥', '保偏稳', '保底']

def compute_gradient_bounds(P, range_up, range_down):
    """
    返回: list of (label, lower_rank, upper_rank)
    lower_rank < upper_rank（位次数值小=排名靠前=更冲）
    """
    rs = GRADIENT_RATIOS['rush_splits']
    ss = GRADIENT_RATIOS['safe_splits']

    bounds = []
    # 冲端3段（位次从小到大 = 从远到近）
    for i in range(3):
        lower = P - int(range_up * rs[i])       # 更冲的一端
        upper = P - int(range_up * rs[i + 1])   # 更稳的一端
        bounds.append((GRADIENT_LABELS[i], lower, upper))

    # 保端3段
    for i in range(3):
        lower = P + int(range_down * ss[i])
        upper = P + int(range_down * ss[i + 1])
        bounds.append((GRADIENT_LABELS[i + 3], lower, upper))

    return bounds

def get_gradient_label(rank, gradient_bounds):
    """根据位次返回梯度标签"""
    for label, lower, upper in gradient_bounds:
        if lower <= rank < upper:
            return label
    # 边界情况：恰好等于最保端
    if rank == gradient_bounds[-1][2]:
        return gradient_bounds[-1][0]
    return '未知'
```

**验证——不同位次学生的梯度边界对比**：

| 段 | P=55000(默认) | P=15000(高分) | P=100000(低分) |
|----|-------------|-------------|---------------|
| range_up/down | 16000/10000 | 4500/3000 | 16000/10000 |
| 高冲 | [39000,45000) | [10500,12188) | [84000,90000) |
| 冲刺 | [45000,50000) | [12188,13594) | [90000,95000) |
| 稳偏冲 | [50000,55000) | [13594,15000) | [95000,100000) |
| 稳妥 | [55000,59000) | [15000,16200) | [100000,104000) |
| 保偏稳 | [59000,62000) | [16200,17100) | [104000,107000) |
| 保底 | [62000,65000] | [17100,18000] | [107000,110000] |

> P=15000时，高冲段仅覆盖位次10500~12188，约1688位，这符合高分段选择稀少的实际情况。每个梯度的宽度自然缩小，但比例结构保持一致。

### 14.11 两层选取架构：锚定专业→院校专业组→组内排序（v4.4核心变更）

#### 14.11.1 问题：选取单位与填报单位不匹配

**现实规则**：四川新高考采用"院校专业组"作为投档单位。一个志愿位 = 一个院校专业组（而非一个专业）。学生选择某个专业组后，需在组内对专业进行排序（通常6个），并决定是否服从调剂。

**v4.3的问题**：系统以"单个专业"为选取单位——每个bin选出1个专业作为志愿。但实际上学生在填报时面对的是"专业组"，组内可能有6~60+个专业，有好有差。如果只看到一个锚定专业而忽略同组其他专业，可能导致：
- 被调剂到组内不理想的专业（"混编组"风险）
- 无法评估"服从调剂"的代价
- 55个志愿位的含义不清——到底是55个专业还是55个组？

**v4.4的解决方案**：引入两层选取架构——

```
外层：选取院校专业组（占用志愿位，≥55个组）
  └─ 触发方式：由"锚定专业"驱动——bin内评分最高的专业触发其所在专业组被选中
  └─ 去重/限频单位改为院校专业组代码(Col3)

内层：组内专业排序（辅助决策，不占志愿位）
  └─ 为每个选中的专业组推荐组内专业排序
  └─ 评估专业组干净度和调剂风险
```

#### 14.11.2 选取流程（外层——以专业组为单位）

```python
def two_layer_selection(candidates_by_bin, student_profile):
    """
    两层选取核心流程：
    外层：按bin选锚定专业 → 映射到院校专业组 → 去重限频
    内层：为每个选中的组做组内排序
    """
    # === 外层：选取院校专业组 ===
    raw_picks = []
    for bin_idx, bin_pool in enumerate(candidates_by_bin):
        if not bin_pool:
            continue
        # Step A: bin内按专业级评分排序（与v4.3一致）
        scored = [(c, score(c, bin_idx/total_bins, student_profile)) for c in bin_pool]
        scored.sort(key=lambda x: -x[1])

        # Step B: 选出评分最高的专业作为"锚定专业"
        anchor = scored[0]
        anchor_major, anchor_score = anchor

        # Step C: 锚定专业所在的院校专业组即为该bin的选中组
        group_code = anchor_major.col3_group_code  # 院校专业组代码

        raw_picks.append({
            'bin_idx': bin_idx,
            'anchor_major': anchor_major,          # 锚定专业（触发选组的那个）
            'anchor_score': anchor_score,
            'group_code': group_code,               # 院校专业组代码
            'school_code': anchor_major.col1_school_code,
            'bin_pool': bin_pool,                    # 保留完整bin候选，供替换用
        })

    # Step D: 去重——按院校专业组代码(Col3)，同一组最多出现1次
    # 注意：不再按院校代码(Col1)去重，因为同校不同组是不同志愿
    picks = deduplicate_by_group(raw_picks, max_same_group=1)

    # Step E: 同校限频——同一院校代码(Col1)在结果中最多出现 max_same_school 次
    picks = school_frequency_limit(picks, max_same_school=2)

    # Step F: 局部限频——连续 locality_window 个bin内同一院校最多1次
    picks = locality_check(picks, locality_window=3)

    # === 内层：组内专业排序 ===
    for pick in picks:
        pick['intra_group_ranking'] = rank_group_majors(
            pick['group_code'], pick['anchor_major'], student_profile
        )
        pick['group_cleanliness'] = assess_group_cleanliness(
            pick['group_code'], student_profile
        )

    return picks
```

**关键设计决策**：

1. **锚定专业驱动选组**：不是先选组再看组内专业，而是先在每个bin内用专业级评分选出最优专业（锚定专业），再展开到其所在的专业组。这保证了"选这个组是因为组里有一个好专业"的逻辑透明性。

2. **去重单位从院校代码(Col1)改为双层**：
   - 第一层：同一院校专业组(Col3) 全局最多出现1次（因为同一个组选两次没有意义）
   - 第二层：同一院校(Col1) 全局最多出现 max_same_school(默认2) 次（保留同校不同组的可能性，如"四川大学01组"和"四川大学03组"可同时入选）

3. **55+的含义**：≥55个不同的院校专业组（即55个不同的志愿位），每个组内推荐专业排序作为参考。

#### 14.11.3 组内专业排序（内层）

```python
def rank_group_majors(group_code, anchor_major, student_profile):
    """
    为选中的专业组做组内专业排序。

    输入：group_code = 院校专业组代码(Col3)
    输出：组内专业按推荐度排序的列表，含推荐/中性/慎选标签

    排序规则（按优先级）：
    1. 锚定专业排第1（因为它是触发选组的原因）
    2. 其余专业按以下评分降序排列：
       a. 专业推荐度：推荐=3, 中性=1, 慎选=0
       b. 学科评估/软科评级：A类=4, B类=3, C类=2, 软科A=2, 无=0
       c. 学生偏好加分（如师范+5）
       d. 25专业最低位次（有独立数据的排前面）
    3. 被学生明确排除的专业标记"⛔排除"但仍展示（提示调剂风险）
    """
    # 获取该组所有专业
    group_majors = get_all_majors_in_group(group_code)

    ranked = []
    for m in group_majors:
        is_anchor = (m.col5_major == anchor_major.col5_major)
        is_excluded = check_exclusion(m, student_profile)
        is_caution = (recommend_score(m) == 0)

        intra_score = (
            recommend_score(m) * 3.0
            + discipline_score(m) * 2.0
            + student_bonus(m) * 1.0
            + (1.0 if m.col28 else 0)  # 有独立位次数据加分
        )

        ranked.append({
            'major_name': m.col5_major,
            'major_class': m.col7_major_class,
            'recommend_tag': '推荐' if recommend_score(m)==3 else ('慎选' if recommend_score(m)==0 else '中性'),
            'discipline_rank': m.col71 or m.col72 or '—',
            'plan_count': m.col17,
            'is_anchor': is_anchor,
            'is_excluded': is_excluded,
            'is_caution': is_caution,
            'intra_score': intra_score,
        })

    # 排序：锚定专业第一，其余按intra_score降序
    ranked.sort(key=lambda x: (-x['is_anchor'], -x['intra_score']))

    # 结果表I列输出格式：取前3个专业名，逗号分隔
    # 示例："计算机科学与技术, 软件工程, 人工智能"
    top3_text = ', '.join([m['major_name'] for m in ranked[:3]])

    return ranked, top3_text
```

#### 14.11.4 专业组干净度评估

**定义**：一个院校专业组的"干净度"描述组内专业质量的一致性。干净的组=组内专业质量均可接受，即使被调剂也不会太差；不干净的组=组内混入了学生明确排除或慎选清单中的专业，调剂风险高。

**数据验证（以王润P=56668为例）**：

| 干净度类型 | 占比 | 说明 |
|-----------|------|------|
| 纯净组 | 35.0% | 组内全部为推荐或中性专业，无慎选 |
| 混编组 | 35.8% | 组内同时含有推荐/中性专业和慎选专业 |
| 劣质组 | 29.2% | 组内全部为慎选或中性，无推荐专业 |

**评分规则**：

```python
def assess_group_cleanliness(group_code, student_profile):
    """
    评估专业组干净度。

    返回: {
        'level': 'clean' | 'mixed' | 'poor',
        'score': float,          # 干净度得分 0~1
        'good_count': int,       # 推荐+中性(非慎选)专业数
        'bad_count': int,        # 慎选专业数
        'excluded_count': int,   # 学生排除的专业数
        'total_count': int,      # 组内总专业数
        'risk_text': str,        # 调剂风险描述
    }
    """
    group_majors = get_all_majors_in_group(group_code)
    total = len(group_majors)

    good = 0    # 推荐或中性
    bad = 0     # 慎选
    excluded = 0  # 学生排除

    for m in group_majors:
        rec = recommend_score(m)
        if check_exclusion(m, student_profile):
            excluded += 1
        if rec == 0:  # 慎选
            bad += 1
        else:
            good += 1

    # 干净度得分 = 好专业占比 - 排除专业惩罚
    cleanliness_score = good / total if total > 0 else 0
    exclusion_penalty = excluded / total if total > 0 else 0
    final_score = max(0, cleanliness_score - exclusion_penalty * 0.5)

    # 分级
    if bad == 0 and excluded == 0:
        level = 'clean'
        risk_text = '组内专业质量一致，调剂风险低'
    elif bad > 0 and good > 0:
        level = 'mixed'
        bad_names = [m.col5_major for m in group_majors if recommend_score(m)==0]
        risk_text = f'混编组，含{bad}个慎选专业（{"/".join(bad_names[:3])}），调剂有风险'
    else:
        level = 'poor'
        risk_text = f'组内{bad}/{total}个专业为慎选，整体质量偏低'

    if excluded > 0:
        ex_names = [m.col5_major for m in group_majors if check_exclusion(m, student_profile)]
        risk_text += f'；含{excluded}个排除专业（{"/".join(ex_names[:2])}）'

    return {
        'level': level,
        'score': round(final_score, 2),
        'good_count': good,
        'bad_count': bad,
        'excluded_count': excluded,
        'total_count': total,
        'risk_text': risk_text,
    }
```

**干净度在评分中的影响**：

干净度不直接参与bin级评分（避免改变锚定专业的选取逻辑），但在以下环节发挥作用：

1. **结果表风险提示**：混编组和劣质组在U列（风险提示）追加调剂风险描述
2. **服从调剂建议**：
   - clean组 → "建议服从调剂"
   - mixed组 → "调剂有风险，注意组内慎选专业：{names}"
   - poor组 → "调剂风险较高，慎重考虑服从调剂"
3. **同分替换参考**：当两个bin的锚定专业评分接近时，优先选干净组的（作为tie-breaker）

```python
def apply_cleanliness_tiebreak(pick_a, pick_b):
    """
    当两个候选评分差距 < 2分时，用干净度打破平局。
    """
    score_diff = abs(pick_a['anchor_score'] - pick_b['anchor_score'])
    if score_diff < 2.0:
        return pick_a if pick_a['group_cleanliness']['score'] >= pick_b['group_cleanliness']['score'] else pick_b
    return pick_a if pick_a['anchor_score'] >= pick_b['anchor_score'] else pick_b
```

#### 14.11.5 去重策略重构（替代14.4/14.5）

v4.4将去重从"院校代码单层去重"改为"专业组+院校双层去重+局部限频"：

```python
def deduplicate_v44(raw_picks, max_same_group=1, max_same_school=2, locality_window=3):
    """
    v4.4 三步去重：
    Step 1: 专业组去重 — 同一院校专业组(Col3)最多出现1次
    Step 2: 院校限频 — 同一院校代码(Col1)最多出现 max_same_school 次
    Step 3: 局部限频 — 连续 locality_window 个bin内同一院校最多1次
    """
    picks = list(raw_picks)

    # --- Step 1: 专业组去重 ---
    group_seen = {}  # {group_code: best_pick}
    step1_result = []
    replacements = []

    for p in picks:
        gc = p['group_code']
        if gc not in group_seen:
            group_seen[gc] = p
            step1_result.append(p)
        else:
            # 同组已有，保留评分更高的
            existing = group_seen[gc]
            if p['anchor_score'] > existing['anchor_score']:
                step1_result.remove(existing)
                replacements.append(existing['bin_idx'])
                group_seen[gc] = p
                step1_result.append(p)
            else:
                replacements.append(p['bin_idx'])

    # 对被替换的bin，选该bin次优候选（不同组）
    for bin_idx in replacements:
        refill_from_bin(bin_idx, step1_result, group_seen, max_same_group)

    # --- Step 2: 院校限频 ---
    school_count = {}
    step2_result = []
    replacements2 = []

    for p in sorted(step1_result, key=lambda x: -x['anchor_score']):
        sc = p['school_code']
        school_count[sc] = school_count.get(sc, 0) + 1
        if school_count[sc] <= max_same_school:
            step2_result.append(p)
        else:
            replacements2.append(p['bin_idx'])

    for bin_idx in replacements2:
        refill_from_bin(bin_idx, step2_result, group_seen, max_same_group, school_count, max_same_school)

    # --- Step 3: 局部限频 ---
    step2_result.sort(key=lambda x: x['bin_idx'])
    for i in range(len(step2_result)):
        code_i = step2_result[i]['school_code']
        for j in range(max(0, i - locality_window + 1), i):
            if step2_result[j]['school_code'] == code_i:
                # 冲突：换当前bin的次优候选（不同院校）
                alt = find_alt_in_bin(step2_result[i]['bin_idx'], exclude_school=code_i)
                if alt:
                    step2_result[i] = alt
                break

    return sorted(step2_result, key=lambda x: x['bin_idx'])
```

**与v4.3的区别**：

| 维度 | v4.3 | v4.4 |
|------|------|------|
| 去重单位 | 院校代码(Col1) | 院校专业组代码(Col3) + 院校代码(Col1) |
| 同一组出现 | 未限制 | 最多1次（同一组选两次无意义） |
| 同一院校出现 | 最多2次 | 最多2次（保留同校不同组的可能） |
| 局部限频 | 同校连续3bin最多1次 | 同校连续3bin最多1次（不变） |
| 去重时机 | bin选最优后 | 两层选取后、组内排序前 |

#### 14.11.6 调剂风险提示生成规则

在v4.3的风险提示基础上，v4.4新增专业组维度的调剂风险提示：

```python
def generate_risk_v44(pick, t, gradient_bounds):
    """
    v4.4风险提示 = v4.3原有风险 + 调剂风险
    """
    risks = generate_risk_v43(pick['anchor_major'], t, gradient_bounds)  # 原有逻辑

    # 追加调剂风险
    gc = pick['group_cleanliness']
    if gc['level'] == 'mixed':
        risks.append(f"混编组({gc['bad_count']}个慎选专业)，慎重调剂")
    elif gc['level'] == 'poor':
        risks.append(f"组内质量偏低({gc['bad_count']}/{gc['total_count']}慎选)，调剂风险高")

    if gc['excluded_count'] > 0:
        risks.append(f"组内含{gc['excluded_count']}个排除专业，务必关注调剂")

    # 组内专业数过多也是风险
    if gc['total_count'] > 20:
        risks.append(f"组内含{gc['total_count']}个专业，调剂范围较广")

    return ' | '.join(risks)
```

**调剂风险展示位置**：结果表W列（风险提示）末尾追加，用" | "分隔符与梯度风险区分。调剂风险部分的字体颜色继承所在行的梯度颜色（不单独着色）。调剂建议的结论性文本（"建议服从调剂"/"慎重调剂"/"调剂风险高"）单独展示在X列。
