## OCR 识别结果的 X 坐标分布

根据实际 OCR 结果分析：

| 内容类型 | X 坐标范围 | 示例 |
|---------|-----------|------|
| 专业代码+名称 | 92-120 | `31捷克语` |
| 计划数 | 510-520 | `1` |
| 收费 | 530-540 | `6000`, `15400`, `免费` |

## 关键问题

### 1. 收费格式多样
- `免费` - 纯文字
- `6000` - 纯数字（元）
- `15400` - 5位数字（元）
- `6875` - 4位数字（元）
- `2免费` - 计划数+免费连在一起

### 2. 计划数和收费的区分
- **计划数**: 通常是 1-2 位数字，X 坐标在 510-520
- **收费**: 通常是 4-5 位数字，X 坐标在 530-540
- **关键**: 用 X 坐标区分，而不是数值大小

### 3. OCR 数字粘连问题
OCR 有时会把 `1 6875`（计划数1，学费6875）识别成单个数字 `16875`。

**解决方案**：对于 5-6 位数字，尝试拆分：
- 5位数字（如 `16875`）：第1位是计划数，后4位是学费
- 6位数字（如 `126000`）：前2位是计划数，后4位是学费
- 学费范围限定在 3000-9999 元

### 4. 跨行专业名称
当专业名称很长时，会跨多行：
```
y=207 | 47外国语言文学类（小语种）（包含专业：日语（日本研究）、法语（法国
y=223 |    研究）、西班牙语（欧洲研究）、意大利语（欧洲研究）、阿拉伯语（中
y=239 |    东学）、波斯语（中东学））（招收英语语种考生）（外语口试）  1  6000
```
- 计划数和收费在最后一行
- 需要合并多行文本

### 5. 专业组计划数
- 专业组行右侧的数字是该组的总计划数
- 示例: `专业组101（再选科目：不限）  7` → 组计划数=7

### 6. 换页处理
- 跨图片时，上下文（考试类型、招生类型、当前院校、当前专业组）需要保持
- 已实现：在 `run_supplementary_ocr` 中传递 `last_exam_type` 和 `last_enrollment_type`

## AI 校验功能

为提高数据准确性，支持 OCR + AI 双重识别模式：

### 工作流程
1. OCR 识别所有图片，得到 OCR 结果
2. AI 视觉模型解析所有图片，得到 AI 结果
3. 比较两者结果，分类为：
   - **matched**: 两者一致的记录
   - **ocr_only**: 仅 OCR 识别到的记录
   - **ai_only**: 仅 AI 识别到的记录
   - **conflicts**: 两者冲突的记录（需人工审核）

### API 端点
- `POST /ocr`: 纯 OCR 模式
- `POST /ocr-with-ai`: OCR + AI 双重识别模式

### 请求参数
```json
{
  "image_urls": ["..."],
  "data_type": "supplementary",
  "year": 2025,
  "enable_ai": true,
  "ai_api_key": "sk-xxx",
  "ai_base_url": "https://api.deepseek.com/v1",
  "ai_model": "deepseek-chat"
}
```

### 响应格式
```json
{
  "total_rows": 100,

  "is_valid": true,
  "errors": [],
  "data": [...],
  "ai_enabled": true,

    }
  },

}
```
