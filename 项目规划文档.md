# 志愿填报助手项目规划文档

## 📋 项目概述

### 项目名称
**志愿填报助手 (VolunteerHelper)**

### 项目目标
为高考和单招学生提供智能化的志愿填报辅助服务，结合 AI 技术帮助学生做出最优的升学决策。

### 目标用户
- **一期**：四川省高考考生
- **二期**：四川单招考生
- **三期**：全国高考考生  全国单招考生

---

## 🎯 核心功能模块

### 1. 数据查询模块
- **院校查询**：按省份、城市、类型、层次筛选
- **专业查询**：按学科门类、就业前景、薪资水平筛选
- **分数线查询**：历年录取分数线、位次查询
- **对比功能**：多所院校/专业横向对比

### 2. AI 智能推荐模块
- **智能匹配**：根据分数、位次、兴趣推荐院校和专业
- **冲稳保策略**：自动生成志愿填报方案
- **风险评估**：录取概率计算
- **个性化推荐**：基于学生画像的精准推荐

### 3. 志愿方案生成模块
- **方案生成**：一键生成完整志愿填报表
- **方案优化**：AI 辅助优化志愿顺序
- **方案保存**：支持多方案对比和保存
- **方案导出**：导出为 PDF/Excel

### 4. 数据分析模块
- **趋势分析**：历年分数线趋势图
- **专业热度**：专业报考热度分析
- **就业分析**：专业就业率、薪资数据
- **地域分析**：不同地区院校分布

### 5. 用户管理模块
- **用户注册/登录**
- **个人信息管理**：分数、位次、选科等
- **VIP 会员系统**：付费增值服务
- **历史记录**：查询历史、方案历史

## 📊 数据库设计

### 核心数据表

#### 1. 院校表 (universities)
```sql
- id: 主键
- name: 院校名称
- code: 院校代码
- province: 所在省份
- city: 所在城市
- type: 院校类型 (综合/理工/师范等)
- level: 院校层次 (985/211/双一流等)
- tags: 院校标签 (JSON)
- created_at, updated_at
```

#### 2. 专业表 (majors)
```sql
- id: 主键
- name: 专业名称
- code: 专业代码
- category: 学科门类
- description: 专业描述
- employment_rate: 就业率
- avg_salary: 平均薪资
- created_at, updated_at
```

#### 3. 招生数据表 (admissions)
```sql
- id: 主键
- university_id: 院校 ID
- major_id: 专业 ID
- year: 年份
- province: 招生省份
- batch: 批次
- plan_count: 计划人数
- min_score: 最低分
- avg_score: 平均分
- max_score: 最高分
- min_rank: 最低位次
- created_at, updated_at
```

#### 4. 用户表 (users)
```sql
- id: 主键
- username: 用户名
- email: 邮箱
- phone: 手机号
- password_hash: 密码哈希
- province: 所在省份
- score: 高考分数
- rank: 位次
- subjects: 选科 (JSON)
- vip_level: VIP 等级
- vip_expire_at: VIP 到期时间
- created_at, updated_at
```

#### 5. 志愿方案表 (volunteer_plans)
```sql
- id: 主键
- user_id: 用户 ID
- name: 方案名称
- year: 年份
- items: 志愿列表 (JSON)
- status: 状态 (草稿/已提交)
- created_at, updated_at
```
